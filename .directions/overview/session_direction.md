# 이번 세션 작업 요청

## 0. 사전 확인 필수

바로 구현하지 말고 아래 문서를 먼저 확인해줘.

1. `.ai-docs/logs/` 폴더에서 가장 최신 세션 요약 문서 확인
2. `.ai-docs/issues/` 폴더에서 가장 최신 이슈 문서 확인
3. `[REVIEW_REQUIRED]` 항목이 있으면 이번 작업과 관련 있는지 먼저 판단
4. 이번 작업이 이전 결정과 충돌하는지 확인

확인 후, 구현 전에 아래 형식으로 짧게 먼저 보고해줘.

```md
## 사전 확인 결과
- 최신 세션:
- 최신 이슈:
- 이번 작업과 관련 있는 미해결 항목:
- 이전 결정과 충돌하는 부분:
- 이번 세션에서 먼저 처리해야 할 순서:
```

---

## 1. 이번 세션의 목표

이번 세션의 목표는 다음과 같아.

* [목표 1]
* [목표 2]
* [목표 3]

이번 세션에서는 위 목표 외의 리팩토링, 디자인 변경, 구조 변경은 하지 마.

---

## 2. 작업 우선순위

아래 순서대로 진행해줘.

### P0: 반드시 먼저 처리

* [가장 중요한 작업]

### P1: P0 완료 후 처리

* [두 번째 작업]

### P2: 시간이 남으면 처리

* [부가 작업]

P0이 해결되지 않으면 P1, P2로 넘어가지 마.

---

## 3. 이번에 바꾸지 말아야 할 것

아래 항목은 이번 세션에서 건드리지 마.

* [건드리면 안 되는 파일/기능/디자인]
* [유지해야 하는 기존 동작]
* [이전 결정 중 유지할 것]

---

## 4. 이전 결정과 달라지는 점

이전 세션 결정과 달라지는 내용이 있다면 여기에 명시한다.

* 기존 결정:
* 이번에 바꾸려는 결정:
* 바꾸는 이유:

만약 이 내용이 기존 코드 구조와 충돌하면, 구현 전에 충돌 지점을 먼저 알려줘.

## 5. 구현 기준

아래 기준을 만족하도록 구현해줘.

* UI에 보이는 텍스트:

* 데이터 저장 방식:
* 기존 데이터와의 호환성:
* 모바일/반응형 고려 여부:
* 접근성 고려 여부:

---

## 6. 완료 조건

작업 완료 후 반드시 아래를 확인해줘.

1. `npm run build` 성공
2. TypeScript 타입 에러 없음
3. 새로 추가한 기능이 실제 화면에서 연결되어 있음
4. dead code가 생기지 않았는지 확인
5. 이번 작업과 관련된 기존 기능이 깨지지 않았는지 확인

---

## 7. 세션 종료 시 문서화

작업이 끝나면 `session_summary.md` 규칙에 따라 아래 문서를 갱신해줘.

1. `.ai-docs/logs/step_[N]_[keyword]_[YYYYMMDD]_routine_adhd.md`
2. 이슈가 있으면 `.ai-docs/issues/issue_step_[N]_[category]_[YYYYMMDD]_routine_adhd.md`

특히 미해결 항목은 반드시 `[REVIEW_REQUIRED]` 태그를 붙여줘.
