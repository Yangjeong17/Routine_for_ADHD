# 기능 추가

## 1. 지원 형식
내보내기, 가져오기 버튼을 눌렀을때, 복사로 작동하는게 아니라 파일로 작동할 수 있도록 만들기.

### 내보내기: 
(기본 값) json 파일로 파일 만들어지게 하며, 자동 다운로드가 아니라 "다운로드" 버튼을 누르면 받아지게 만들기. 
(optional) 그리고 다운로드 버튼 옆에는 "텍스트 복사하기" 기능을 넣어서 지금처럼 텍스트로 된 영역을 표시하여 사용자가 복사할 수 있도록 하며, Copy 버튼(코드를 복사할때 주로 쓰는 그 버튼)을 만들어서 사용자가 한번에 복사할 수 있도록 구현하기.

### 가져오기: 
(기본 값) json, txt, docx 파일 등을 바로 드래그앤 그랍할 수 있게 만들기(PC). 추후 App에서는 "파일 선택" 버튼을 누르면 폰에 있는 개체선택 연결이 바로 되어, 폰에 저장된 문서를 선택할 수 있도록 만듦
(optional) 그리고 파일선택 버튼 옆에는 "텍스트 붙여넣기" 기능을 넣어서 지금처럼 텍스트로 된 영역을 표시하여 사용자가 붙여넣기 할 수 있도록 구현하기.



## 2. 전체 틀
시간 순으로 블록이 나눠져 있긴 하지만 전체적으로 시간이 서로 다른 블록이 같은 선 상에 있기 때문에 헷갈림. 

### 전체 틀을 시간 순으로 나누고 가장 왼쪽에 시간을 넣어줌. 시간 단위는 24시간으로 넣어주며, 시작은 오전 6시로 보이되, 스크롤 하면 그 전의 시간도 선택 할 수 있어야 함(안보이는 스크롤 구현).

### 시간에 따라 움직이는 얇은 빨간 실선을 두어 현재 시간이 몇시인지 직관적으로 볼 수 있게 구현(한국기준)

## 3. 수정가능 구현
"수정" 버튼을 누르면 루틴의 모든 내용 수정이 가능하긴 하지만, 직관적이지 않다고 생각됨. 각 부분에 맞는 수정기능 구현이 필요.

### 제목
제목 옆에는 펜 표시의 버튼을 두어 이를 누르면 현재처럼 루틴 블록을 수정할 수 있도록 구현


### 요일, 시간, 삭제
드래그로 블록을 이동 및 삭제가 가능하도록 구현
삭제의 경우 하단으로 계속 드래그하면 시각적 피드백(빨간 투명 하이라이트)을 보여서 삭제 가능함을 알려줌

### 카테고리, 중요도
선택하면 드랍박스 보이고 이 중 선택 할 수 있도록 구현


## 4. 구현 순서
바로 구현하지 말고 Spec을 새로 만들어서 체계적으로 진행해주세요.

이번 변경은 단순 기능 추가가 아니라 기존 주간표 UI를 시간축 기반 레이아웃으로 바꾸는 리팩토링이 포함되어 있으므로, 먼저 요구사항과 구현 단계를 명확히 나누고 싶습니다.

우선순위는 다음과 같이 잡아주세요.

1. 파일 기반 Import/Export

   * JSON 파일 다운로드
   * JSON 파일 드래그앤드롭 가져오기
   * 파일 선택 가져오기
   * 선택적으로 텍스트 복사/붙여넣기 유지

2. 타임라인 기반 주간 레이아웃

   * 좌측 24시간 시간 라벨
   * 월~일 상단 고정 헤더
   * 각 루틴 블록을 start/end 기준으로 시간축 위에 배치
   * 기본 화면은 오전 6시부터 보이게 초기 스크롤
   * 위로 스크롤하면 0~5시도 볼 수 있게 구현
   * 한국 시간 기준 현재 시간 빨간 실선 표시

3. 인라인 편집 개선

   * 제목 옆 펜 아이콘으로 기존 수정 모달 열기
   * 카테고리와 중요도는 블록에서 바로 드롭다운으로 수정
   * 삭제는 기존 삭제 버튼 유지

4. 드래그 편집은 이번 단계에서 바로 구현하지 말고, 타임라인 레이아웃이 안정화된 뒤 다음 단계로 분리해주세요.

   * 요일 간 이동
   * 시간 변경
   * 드래그 삭제
   * 모바일 터치 대응
     이 부분은 별도 Spec으로 나중에 진행하고 싶습니다.

# 피드백

추가 정책을 반영해서 Spec을 작성해주세요.

## 1. 지원 파일 형식

Import는 `.json`만으로 제한하지 않고, 단계적으로 확장 가능한 구조로 설계해주세요.

### 이번 단계에서 우선 지원

* `.json`
* `.txt`

### 다음 단계에서 지원할 수 있도록 구조 고려

* `.docx`
* `.md`
* `.pdf`

파일별 처리 정책은 다음과 같습니다.

* `.json`: Routine_Data JSON으로 직접 파싱
* `.txt`: 텍스트 내용을 읽고, 내용이 JSON이면 JSON 파싱. JSON이 아니면 추후 텍스트 루틴 파싱 기능으로 확장 가능하게 설계
* `.docx`: 이번 단계에서 반드시 구현하지 않아도 되지만, 추후 mammoth.js 같은 라이브러리로 텍스트를 추출한 뒤 JSON 또는 루틴 텍스트로 파싱할 수 있도록 Import 구조를 확장 가능하게 설계

## 2. 시간 단위 정책

시간 라벨과 실제 입력 가능 시간 단위를 분리해주세요.

* 좌측 시간 라벨은 1시간 단위로 표시
* 배경 보조선은 30분 단위로 표시
* 루틴 블록의 위치 계산은 5분 단위까지 반영
* 수정 모달에서는 시작 시간/종료 시간을 5분 단위로 선택 가능
* 화면에 5분 단위 라벨은 표시하지 않음
* 사용자가 세부 시간을 선택하면 블록 위치는 해당 시간에 맞춰 이동함

## 3. 짧은 블록 표시 정책

5분, 10분, 15분짜리 짧은 루틴도 사용할 수 있어야 합니다.

* 실제 duration_minutes 값은 그대로 유지
* UI에서는 클릭 가능성을 위해 최소 표시 높이를 둠
* 최소 표시 높이는 20분 분량 또는 최소 28px 중 더 큰 값으로 설정
* 짧은 블록은 기본적으로 제목만 표시
* 마우스를 올리거나 키보드 focus를 받으면 블록이 살짝 확장되거나 popover로 상세 정보를 표시
* 모바일에서는 hover 대신 tap으로 상세 popover를 표시
* 블록 주변에 보이지 않는 hit area를 두어 선택 가능 영역을 확장
* hit area는 최소 상하 6~10px 정도로 설정

## 4. 블록 겹침 처리

같은 요일 안에서 시간대가 겹치는 블록은 물리적으로 겹치지 않게 가로로 나란히 배치해주세요.

* 겹치는 블록은 개수에 따라 width를 줄이고 left 위치를 조정
* 2개 겹침: 각 50% 너비
* 3개 겹침: 각 33% 너비
* 블록이 좁아져서 모든 정보를 표시하기 어려우면 제목만 표시
* 중요도는 배지보다 블록 색상으로 표현
* 시간, 카테고리, 메모 등 상세 정보는 hover 또는 tap 시 popover로 표시

## 5. 카테고리 편집

카테고리 드롭다운은 "기존 카테고리 목록에서 선택 + 직접 입력" 형태로 구현해주세요.

* 기존 카테고리 목록은 현재 루틴 데이터의 category 값을 기준으로 자동 생성
* 사용자는 기존 카테고리를 선택할 수 있음
* 사용자는 새 카테고리를 직접 입력할 수 있음
