[System Directive]

Role: Technical Project Manager

Task: 현재 세션의 작업 내용을 다음 AI가 즉시 이어받을 수 있도록 세션 요약 문서와 이슈 문서를 생성한다.

---

# 1. 공통 규칙

## 1.1 목적

이 문서의 목적은 단순 기록이 아니라, 다음 세션 AI가 아래 내용을 빠르게 파악하도록 만드는 것이다.

1. 이번 세션에서 실제로 무엇이 바뀌었는지
2. 어떤 결정이 확정되었는지
3. 어떤 문제가 남아있는지
4. 다음 세션에서 무엇을 먼저 해야 하는지
5. 빌드와 동작 검증이 완료되었는지

---

## 1.2 저장 위치

세션 요약 문서:

```txt
.ai-docs/logs/
```

이슈 문서:

```txt
.ai-docs/issues/
```

---

## 1.3 파일명 규칙

모든 파일명은 영문 소문자와 언더스코어만 사용한다.

세션 요약:

```txt
step_[N]_[keyword]_[YYYYMMDD]_routine_adhd.md
```

예시:

```txt
step_7_drag_edit_20260615_routine_adhd.md
```

이슈 문서:

```txt
issue_step_[N]_[category]_[YYYYMMDD]_routine_adhd.md
```

예시:

```txt
issue_step_7_drag_edit_20260615_routine_adhd.md
```

주의:

* 한글 파일명 사용 금지
* 대문자 사용 금지
* `Session_` 접두어 사용 금지
* 기존 문서에 다른 규칙이 섞여 있어도 앞으로는 이 규칙을 따른다

---

# 3. 이슈 문서 작성 규칙

작업 중 문제가 발생했으면 반드시 별도 이슈 문서를 생성한다.

```md
**저장 경로 및 파일명: .ai-docs/issues/[filename].md**

# Issue Log - Step [N]: [이슈명]

## 1. Resolved Issues

### 1.1 [이슈 제목]

- 증상:
- 원인:
- 해결 방법:
- 수정 파일:
- 검증 결과:

## 2. Unresolved Issues

### [REVIEW_REQUIRED] 2.1 [이슈 제목]

- 현재 상태:
- 시도한 방법:
- 실패 원인:
- 다음 세션에서 확인할 것:
- 관련 파일:
- 우선순위: P0/P1/P2

## 3. Carry-Over Checklist

다음 세션 시작 시 반드시 확인해야 할 항목이다.

```

# 4. 세션 종료 전 필수 확인

문서 작성 전 아래를 확인한다.

1. `npm run build` 실행 여부
2. 실제 화면에 수정 사항이 연결되었는지 여부
3. dead code 발생 여부
4. 기존 기능 회귀 여부
5. `[REVIEW_REQUIRED]` 항목 존재 여부
6. 다음 세션에서 가장 먼저 해야 할 작업

검증하지 못한 항목은 “성공”이라고 쓰지 말고 “미실행”으로 기록한다.
